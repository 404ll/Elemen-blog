因为一直有在关注 agent 相关的博文和帖子，所以对一些名词有大概的了解（MCP、Skill、harness 等等），所以在正式开始学习 agent 前，花了一点时间让 AI 帮我理了一下这些名词之间的联系以及演化原因。
这里想介绍一下我自己的学习方法：使用**「第一性原理」**去理解一个概念/范式出现的根本原因，然后用流程图去梳理每个概念之间的关系，因为一切的演化都是有理由的。这样下来其实会对一些东西建立自己的理解，而不是干瘪的背诵概念。

回到主题，因为我没有 Python 基础，处于能看懂基本逻辑但是一些细节看不懂的情况，但是我又这段时间比较忙，没有办法静下心来好好学语法，因此在整个学习过程中，**我没有去看代码的具体实现逻辑，只是学习了设计思想和架构**。
同时和 AI 讨论，去设计自己的写作 agent，这个过程可以体验从一个固定代码控制「死流程」到拥有自己决策和思考「自主 agentic」的演变过程。

至于收获，我认为是对 agent 的理解会更深更清晰了（对比前段时间的自己），但是真正动手实现一个生产级的 agent 肯定是不行的，后续应该会以业务驱动落地一些真正可用的 agent 吧，过程中也学习一下后端相关的知识。

<Callout type="info" title="核心理念">
  **「将你的思考和实现外包给 agent，将理解留给自己」**
</Callout>

随着 agent 的发展，人为参与的节点会不断地往高层抽象，从最开始的审核代码，到后面审核方案，再到现在规定 spec，确定 agent 的能力边界。

**学习 agent 是为了更好的使用 agent。**
